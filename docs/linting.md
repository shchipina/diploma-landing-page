## 1. Обрані інструменти
* **ESLint:** Використовується для пошуку логічних помилок та дотримання best-practices React
* **Prettier:** Використовується для жорсткої фіксації стилю коду (форматування), інтегрований в ESLint
* **TypeScript (tsc):** Забезпечує статичну типізацію

## 2. Базові правила
* `prettier/prettier`: Всі відхилення від стилю (лапки, відступи) викликають помилку
* `no-explicit-any`: Забороняє використання типу `any`, вимагаючи чіткого опису інтерфейсів
* `no-console`: Попереджає про залишені в коді `console.log`

## 3. Інструкції з запуску
* `npm run lint` — перевірка коду лінтером
* `npm run lint:fix` — автоматичне виправлення проблем форматування
* `npm run typecheck` — перевірка статичних типів без компіляції (`tsc --noEmit`)
* `npm run check-all` — комплексна перевірка (типи + лінтер)

## 4. Git Hooks (Husky)
У проєкті налаштовано pre-commit хуки. Перед створенням кожного коміту автоматично запускається перевірка staged-файлів (через `lint-staged`). Якщо знайдено помилки, коміт буде відхилено до їх виправлення. Це гарантує чистоту гілки `main`

## 5. Інтеграція з процесом збірки
Скрипт збірки проєкту налаштований так: `"build": "npm run check-all && vite build"`, означає, що CI або деплой на GitHub Pages автоматично впаде, якщо код не пройде перевірку типів або лінтера
